================================================================================
TROUBLESHOOTING
================================================================================

Issue: "ModuleNotFoundError"
Solution: pip install -r requirements.txt

Issue: "Port 5000 already in use"
Solution: Change port in app.py or kill process using port 5000

Issue: "Database file not found"
Solution: Database files are auto-created. Check permissions.

Issue: Chat not responding
Solution: Open browser console (F12) and check for errors

Issue: Login not working
Solution: Clear browser cache/cookies and try again

================================================================================
DEPENDENCIES INSTALLATION
================================================================================

If pip install fails:

1. Ensure Python 3.7+ is installed
   python --version

2. Upgrade pip
   python -m pip install --upgrade pip

3. Install each package individually if batch install fails
   pip install Flask==2.3.3
   pip install Flask-CORS==4.0.0
   pip install bcrypt==4.0.1
   pip install Werkzeug==2.3.7

DEVELOPMENT TIPS
================================================================================

1. Debug Mode:
   Set FLASK_ENV=development (Windows)
   export FLASK_ENV=development (Mac/Linux)

2. Modify Intent Recognition:
   - Edit nlp/intents.py to add new intents
   - Update api/chat.py to handle new intents

3. Add New Books:
   - Edit database/books.json
   - Restart the application

4. Customize Styling:
   - Edit static/css/login.css or home.css
   - Changes take effect immediately

5. API Testing:
   - Use Postman or curl
   - All endpoints use JSON
   - Login required for most endpoints

================================================================================
PRODUCTION DEPLOYMENT
================================================================================

Before deploying to production:

1. Change SECRET_KEY in config.py
2. Set DEBUG = False in config.py
3. Use a proper WSGI server (Gunicorn/uWSGI)
4. Enable HTTPS
5. Set up proper logging
6. Configure database backups
7. Implement rate limiting
